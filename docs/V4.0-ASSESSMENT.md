## 2. 能力级别评定

### 2.1 评级标准

| 级别 | 名称 | 标准 |
|------|------|------|
| L1 | 玩具 | 单文件脚本，无测试 |
| L2 | 原型 | 基本可跑，少量测试 |
| L3 | 业余生产级 | 有测试、有文档、核心功能完整 |
| L4 | 工业生产级 | 完整工具链、高测试覆盖、可团队协作 |
| L5 | 商业级 | 有 SLA 保障、多平台支持、企业特性 |
| L6 | 行业领先 | 在某个领域超过主流竞品 |
| L7 | 标杆 | 定义行业标准 |

### 2.2 V4.0 评级

#### 通用编码能力：**L4 工业生产级**（V3.0 为 L3）

提升依据：
- ✅ Git 工具集（5 个工具）— 补齐 Claude Code 对标的最后一块短板
- ✅ Diff 预览 — edit_file/multi_edit 前置展示变更
- ✅ 多文件原子编辑 — 两阶段事务 + 自动回滚
- ✅ 子 Agent 编排 — SubAgentOrchestrator + 并行 + 超时
- ✅ 观测性指标 — MetricsCollector 线程安全单例 + 3 类指标
- ✅ 完整文档 — README + USER_GUIDE + ARCHITECTURE
- ✅ 测试覆盖 73.9%（业界标准 >70%）
- ✅ 1323 个测试，0 回归

#### 嵌入式专属能力：**L5 商业级**（保持领先）

V4.0 未削减任何嵌入式能力，且通过 tree-sitter 引导降低了使用门槛：
- ✅ EmbedGuard 静态分析（STM32/MISRA-C 专属）
- ✅ tree-sitter C/C++ 代码索引（一键安装）
- ✅ STM32CubeMX + PlatformIO + Keil MDK 集成
- ✅ HAL 外设规则库（G4/F4 系列）
- ✅ 远程 SSH 烧录/调试
- ✅ OS 沙箱（Linux bwrap / Windows 路径校验）

#### 综合评级：**L4+ 工业生产级（嵌入式领域 L5）**

## 6. 结论

### 6.1 V4.0 达成情况

V4.0 的六大 Track（Git 工具 / Diff 预览 / MultiEdit / 子 Agent / 观测性 / tree-sitter 引导）全部完成，1323 个测试通过，0 回归。

**通用编码能力从 L3 提升到 L4**，补齐了与 Claude Code 对标的所有 P0/P1 短板（Git 工具、Diff 预览、多文件编辑、子 Agent）。

**嵌入式能力保持 L5 领先**，且通过 tree-sitter 一键引导降低了使用门槛。

### 6.2 V5.0 方向

V5.0 将聚焦三个方向：
1. **通用能力深化**：Git 高级命令 + 工具权限白名单
2. **嵌入式能力扩展**：RTOS 支持 + 多 MCU 平台
3. **工程化基础设施**：CI/CD + 性能基准 + 覆盖率门禁

预计 V5.0 完成后，Iron CLI 将达到 **L4+ 通用 / L5+ 嵌入式** 的综合水平，在嵌入式 AI Agent 领域保持行业领先。

### 6.3 发布建议

V4.0 已满足发布条件：
- ✅ 全量测试通过（1323 passed, 0 failed）
- ✅ 无 P0 问题

- ✅ 特性门控完备（5 个新特性均可关闭）

**建议标记为 `4.0.0-release`**，V5.0 作为下一个里程碑版本规划。
